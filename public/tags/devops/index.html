折腾github action的一天

了解github action后，第一步想到的是改造下这个博客。利用github action的工作流，做到直接在github上编辑markdown，保存后，触发一个设定的workflow，做到编译，部署编译好的静态网站到GitHub pages。 这里面关键是，如何部署编译好的静态网站到Github Pages。我用的是hexo这个静态网站生成器，代码是在一个独立repo，与要部署的目标repo不同。在GitHub的runner也就是服务器上，如何设置github credentials，从而能从那里部署是关键。 几番搜索发现了有个现成的github action: hexo-action。简单总结下: Generate a deploy key pair. ssh-keygen -t rsa -C "username@example.com" Public key is saved in github page repo’s Settings/Deploy Keys. Private key is saved as a secret in source repo’s Settings/Secrets, e.g., named DEPLOY_KEY.
